# P2P 文件传输客户端 — 需求文档 (Qt GUI)

## 一、项目概述

基于已实现的 Registry Server，开发 P2P 文件传输客户端。客户端向 Registry Server
注册自身信息后，可以从其他注册节点拉取文件或向其他节点推送文件。
文件传输在局域网内通过 TCP 直连完成。**GUI 使用 Qt**，左右分栏设计，参考网易UU远程的文件传输界面（uuTrans.png）。

---

## 二、整体流程

```
┌──────────┐         ┌──────────────┐         ┌──────────┐
│  Client A │         │ Registry     │         │  Client B │
│(Win11)    │         │ Server       │         │(Ubuntu VM)│
└────┬─────┘         └──────┬───────┘         └────┬─────┘
     │                      │                      │
     │  ① 用户点击「注册」                              │
     │  ② connect + getsockname() 获取本机 LAN IP     │
     │                      │                      │
     │  ③ register ────────→│                      │
     │  ④ register_ack ←────│                      │
     │     (含所有 peers)    │                      │
     │                      │                      │
     │  ⑤ 用户点击「刷新对端」                           │
     │  ⑥ query ───────────→│                      │
     │  ⑦ query_ack ←───────│                      │
     │     (含 B 的信息)     │                      │
     │                      │                      │
     │  ⑧ 选择 B + 选择本地文件                          │
     │  ⑨ 点击发送 ───────────────────────────────→ ⑩ B 接收
     │     (TCP 直连 B 的 P2P 端口)                  │
     │                      │                      │
     │  ⑪ 用户点击「注销」                               │
     │  ⑫ unregister ──────→│                      │
     │  ⑬ unregister_ack ←──│                      │
```

## 三、启动参数

```
./client [p2p_port]
```

| 参数 | 说明 |
|------|------|
| p2p_port | 可选，本机 P2P 文件传输监听端口，默认随机分配 |

- Registry Server IP 和 P2P IP：客户端**自动获取**（`getsockname()`）
- Registry Server 端口：默认 `8888`，在 GUI 中可修改
- 节点名称：默认读取本机 hostname，在 GUI 中可修改

启动后显示 GUI 窗口，**不会自动注册**，需要手动点击「注册」按钮。

## 五、交互操作

### 5.1 注册

- 在配置栏填写 Server IP（手动填写，如 `192.168.1.100`）、Server Port（默认 8888）
- 填写本机名称（默认读取 hostname）、P2P 端口（默认随机分配）
- 点击「**注册**」按钮：
  1. 连接 Registry Server，在已连接的 socket 上调用 `getsockname()` 获取本机 LAN IP
  2. 发送 register 请求（`ip` 为自动获取的 LAN IP，`port` 为配置的 P2P 端口，`name` 为配置的名称）
  3. 收到 register_ack 后，左侧对端列表显示其他节点（**排除自身**），标题栏更新名称
- 注册成功后，「注册」按钮禁用，「注销」按钮启用
- 未注册时，对端列表为空且「发送 →」按钮禁用

### 5.2 注销

- 点击「**注销**」按钮：
  1. 发送 unregister 请求到 Registry Server
  2. 收到 unregister_ack 后，清空对端列表，标题栏恢复「未注册」
- 「注册」按钮恢复可用，「注销」按钮禁用

### 5.3 刷新对端

- 点击「**刷新对端**」按钮：
  1. 发送 query 请求到 Registry Server
  2. 收到 query_ack 后，更新左侧对端列表（**排除自身**，按 ip:port 匹配）
- 已注册状态下可用，未注册时禁用
- 不会自动定时刷新，完全手动触发

### 5.4 对端选择

- 左侧显示所有已注册的其他节点
- 显示格式：`[name]  [ip:port]`
- 选中一个节点后，「发送 →」按钮变为可用
- 未注册时列表为空，显示提示「未注册，请点击上方注册按钮」

### 5.5 文件/目录选择

- 右侧 QTreeView 绑定 QFileSystemModel，显示本机文件系统
- 支持 **Ctrl+Click** 多选文件和目录（可以混合选择）
- 「选择文件」按钮：弹出 QFileDialog，支持多选文件
- 「选择文件夹」按钮：弹出 QFileDialog，选中一个目录
- 多次选择会追加到已选列表，不覆盖之前的选择

### 5.6 发送

- 点击「发送 →」按钮
- 选中的文件/目录加入底部传输队列，自动开始排队传输
- 排队规则：**串行传输，一个文件传完再传下一个**
- 文件夹递归展开为文件列表，按深度优先顺序排队
- 实时更新每个文件的传输进度条

### 5.7 传输队列

| 列 | 内容 |
|----|------|
| 文件名 | 文件路径 |
| 目标 | 对端节点名称 |
| 大小 | 格式化后的大小（KB/MB/GB） |
| 进度 | QProgressBar + 百分比文字 |
| 状态 | 排队中 / 传输中 / 完成 / 失败 |

- 已完成的项目保留显示 3 秒后从队列中移除
- 失败的项目保留显示，鼠标悬停显示错误原因

## 六、P2P 文件传输协议

### 6.1 传输模型

推送方 Client A（主动方）→ TCP connect → 接收方 Client B（被动方，listen 在 P2P 端口）

每次「发送」操作建立**一条 TCP 连接**，所有排队文件串行传输，传完断开。

### 6.2 消息格式（与 Registry Server 一致）

```
┌──────────────────────┬────────────────────────┐
│  4 bytes (大端序)     │     N bytes            │
│  负载长度 N            │     UTF-8 JSON         │
└──────────────────────┴────────────────────────┘
```

### 6.3 推送传输协议

**阶段1：握手**

```
推送方 → 接收方:
  {"type":"push_hello","count":3}

接收方 → 推送方:
  {"type":"push_ready"}
```

**阶段2：文件逐个排队传输**

每个文件按照：`file_start` → `file_data`(n次) → `file_end` 的顺序。

```
推送方 → 接收方:
  {"type":"file_start","path":"photos/a.png","size":3500000}

  {"type":"file_data","offset":0,"data":"<base64>"}
  {"type":"file_data","offset":65536,"data":"<base64>"}
  ...

  {"type":"file_end","path":"photos/a.png","status":"ok"}
```

- `data` 字段为文件二进制数据块的 **base64 编码**
- 块大小 64KB
- 发送完 file_end 后等待对端 file_ack，再发下一个文件

```
接收方 → 推送方（每收到一块）:
  {"type":"ack","path":"photos/a.png","offset":65536}
```

**阶段3：结束**

```
推送方 → 接收方:
  {"type":"transfer_done","success":3,"failed":0}

接收方 → 推送方:
  {"type":"bye"}
```

- 传输完成后双方关闭此 TCP 连接

## 七、文件夹传输规则

- 文件夹递归展开为文件列表，保持目录结构
- 展开顺序：深度优先
- 所有文件共用一个 TCP 连接串行传输
- 接收方自动创建对应目录层级
- 空目录不传输

示例：选择 `/home/user/docs/`（内含 `a.txt` 和 `sub/b.txt`）

`push_hello.count` = 2，展开队列：
```
docs/a.txt
docs/sub/b.txt
```

接收方在默认路径下创建：
```
docs/a.txt
docs/sub/b.txt
```

## 八、接收方默认存储路径

```
Windows:  %USERPROFILE%\P2P_Received\
Linux:    ~/P2P_Received/
```

可在设置中修改。收到的文件保持原路径结构。

## 九、错误处理

| 情况 | 行为 |
|------|------|
| 注册时 Registry Server 不可达 | 弹窗提示「无法连接 Registry Server」，不注册 |
| 注册失败（Server 返回错误） | 弹窗显示错误原因 |
| 注销时网络断开 | 静默处理，本地清空状态 |
| 对端不在线（TCP connect 失败） | 队列中标记为「失败」，提示 `对端离线` |
| 传输中途连接断开 | 当前文件「失败」，已传输部分删除，不继续下一个 |
| 文件不存在/无权限 | 跳过当前文件，标记「失败」，继续下一个 |
| 本地磁盘空间不足 | 当前文件「失败」，提示 `磁盘已满` |
| 未注册时点击刷新/发送/注销 | 按钮禁用，不会触发 |
| 窗口关闭 | 如果已注册则自动注销，等待传输完成后退出 |
| 未选择对端就点发送 | 提示「请先选择目标节点」 |
| 未选择文件就点发送 | 提示「请先选择要发送的文件」 |

## 十、技术实现要点

| 模块 | 说明 |
|------|------|
| 语言 | C++17 + Qt 6 |
| 构建 | CMake + FetchContent（复用 nlohmann/json） |
| Registry 通信 | 独立 TCP 长连接，register / query / unregister 复用同一连接 |
| 本机 IP | `getsockname()` 自动获取，无需用户手动填写 |
| P2P 监听 | 独立线程 accept 在 p2p_port |
| 文件传输连接 | 每次「发送」新建 TCP → 串行传完 → 关闭 |
| UI | QMainWindow + QSplitter 左右分栏 |
| 配置栏 | QLineEdit + QPushButton，水平布局 |
| 文件系统树 | QFileSystemModel + QTreeView |
| 目录递归 | std::filesystem |
| 传输线程 | QThread（与 UI 线程通过信号槽通信） |
| 进度更新 | 传输线程 emit → UI 线程更新进度条 |
| 对端过滤 | query 结果中排除自身（按 ip:port 匹配） |
| 分块大小 | 64KB |
| base64 | Qt 内置 QByteArray::toBase64() / fromBase64() |
